Unsupervised New Physics detection at 40 MHz

In this challenge, you will develop algorithms for detecting New Physics by reformulating the problem as an out-of-distribution detection task. Armed with four-vectors of the highest-momentum jets, electrons, and muons in a collision event, together with the missing transverse energy, the goal is to pick up New Physics signatures from the overwhelming Standard Model background, using anomaly detection.

The algorithms are intended for a "triggering" environment, meaning both resources and time is limited. Therefore only four-vectors are provided for the leading 10 jets, 4 muons, 4 electrons and the missing ET.

Further complicating things, is that trigger algorithms must have a very low latency, of around one microsecond, and in addition must be resource efficient. The task is therefore to find the architecture that maximises the sensitivity for New Physics, but at the lowest possible resource budget (computed in FLOPs). The winner will be the team with the highest true positive rate at a fixed false positive rate of 10E-5, roughly the SM suppression needed for a trigger algorithm, and the lowest possible number of FLOPs.

Datasets and examples

A link to the data is given below. We provide a cocktail of Standard Model events intended for training. In addition, we provide four different Beyond Standard Model signals one can test models on. Finally, a "blackbox" test sample, containing a mixture of SM and an unknow New Physics signal is provided and will be further described in the section below on how to upload a contribution.

How to submit a contribution

In order to participate in the challenge, you need to evaluate your model on the blackbox dataset, containing a mixture of Standard Model and New Physics events. Each team will be asked to submit a list of events passing a selection on the algorithm as well as the total number of FLOPs for the architecture. The submission should be a .npy file (with event IDs) and a .txt file (with number of floating point operations). The name of the file should be unique and correpond to a participant/group name.
